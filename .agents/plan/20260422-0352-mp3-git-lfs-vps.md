# Fixed Plan: ~/server/mozaic.fm に mp3 を LFS で置ける repo を作る

## tldr

- 正本は `~/server/mozaic.fm.git` (bare) と `~/server/mozaic.fm` (clone) の 2 つで、`origin` は VPS 上の bare repo を `ssh://` URL で参照する
- 作業木の実体は `~/server/mozaic.fm` に置き、`~/develop/mozaic.fm` から symlink する (既存 `jxck.io` と同じ topology)
- LFS の pure-SSH transfer に必要な `git-lfs-transfer` は git-lfs 本体に含まれない別プロジェクト (`charmbracelet/git-lfs-transfer`) なので、mise の `go:` backend で別途入れる
- mise の shims と `MISE_GLOBAL_CONFIG_FILE` を `~/.zshenv` で non-interactive shell にも降ろさないと、SSH 越しの `git-lfs-transfer` invocation が解決できない
- LFS 対象は `files.mozaic.fm/` 配下に限定。`github` mirror には `remote.github.lfsurl` を VPS に向けて mp3 実体を流出させない

## Plan by Codex (executed by Claude)

### ゴール

以下の 4 点を同時に満たす構成を作る。

- `~/server/mozaic.fm.git`: bare repo。Git の正本 remote 兼 LFS オブジェクト保管先
- `~/server/mozaic.fm`: 日常作業用 clone (実体)。repo root に `feed.mozaic.fm` / `files.mozaic.fm` / `www.mozaic.fm` を持つ
- `~/develop/mozaic.fm`: `~/server/mozaic.fm` への symlink。日常的なエディタ作業はこのパスから行う
- `github.com/Jxck/mozaic.fm`: 公開用 mirror。repo 全体の commit は置くが、mp3 実体は持たせない

今回の範囲は「repo の器を作ること」と「`files.mozaic.fm/` 配下の mp3 1 本で transport を検証すること」までに留める。

### 範囲外

- `files.mozaic.fm/` への全 mp3 一括取り込み
- `feed.mozaic.fm/` / `www.mozaic.fm/` の中身の移植
- `jxck.io` 側 `mozaic.fm/` 実装の削除
- h2o 設定変更
- バックアップ運用の実装

### 設計方針

#### repo と remote の役割

- `~/server/mozaic.fm` を repo root の実体とし、`~/develop/mozaic.fm` から symlink する
- `origin = ssh://jxck@jxck.io/home/jxck/server/mozaic.fm.git`
  - Git の push/pull の正本
  - LFS upload/download の正本
  - **必ず `ssh://` URL を使う**。SCP-style (`user@host:path`) は git-lfs の pure-SSH transfer 経路に乗りにくい
- `github = git@github.com:Jxck/mozaic.fm.git`
  - repo 全体の commit を載せる mirror
  - GitHub 側の LFS endpoint は使わない

#### LFS の source of truth

- LFS 対象は `files.mozaic.fm/` 配下だけに限定する
- `.gitattributes` もその前提で管理する
- `*.mp3` のような repo 全体指定にはしない

#### LFS endpoint の持ち方

- `.lfsconfig` は作らない
- `origin` は `ssh://` URL から導出される pure-SSH transfer をそのまま使う
- `github` は `.git/config` の `remote.github.lfsurl` / `remote.github.lfspushurl` を VPS に向ける
  - これで `git push github main` 時も LFS 実体は VPS に送られ、GitHub には流れない
  - 公開 repo 側には VPS の LFS endpoint を commit しない

#### server 側の前提

- bare repo 単体でよい
- ただし non-interactive SSH から **`git-lfs` と `git-lfs-transfer` の両方が見える必要がある**
  - `git-lfs` は Homebrew で入る
  - `git-lfs-transfer` は git-lfs 本体には含まれない。`charmbracelet/git-lfs-transfer` を別途入れる必要がある
- 最初の LFS push 前に `ssh jxck@jxck.io 'which git-lfs && which git-lfs-transfer'` を確認する

### 手順

#### 1. `git-lfs` を入れる

Brewfile に `brew "git-lfs"` を追加して `brew bundle`。

```bash
git lfs version    # 3.x 系が出れば OK
```

必要条件は「pure-SSH transfer をサポートする `git-lfs` 3.x 系が入ること」であり、特定 patch version は前提にしない。

#### 2. `git-lfs-transfer` を入れる

`git-lfs-transfer` バイナリは git-lfs プロジェクト本体には含まれない。`charmbracelet/git-lfs-transfer` (別プロジェクト) から取得する。Homebrew formula は無い。release バイナリは古いことがあるので、mise の `go:` backend で main HEAD から install する方針を採る。

`~/dotfiles/.mise.toml` (= `MISE_GLOBAL_CONFIG_FILE` の指す global config) に追加:

```toml
[tools]
"go:github.com/charmbracelet/git-lfs-transfer" = "main"
```

```bash
mise install
ls ~/.local/share/mise/shims/git-lfs-transfer    # shim ができていること
```

#### 3. mise shims と `MISE_GLOBAL_CONFIG_FILE` を non-interactive shell に降ろす

pure-SSH transfer は client が `ssh user@host git-lfs-transfer ...` を発行するため、**non-interactive shell の PATH に `git-lfs-transfer` が見える** 必要がある。mise の通常の `mise activate` は interactive shell 用 hook なので別途設定する。

`~/.zshenv` に以下を追加 (interactive 側は `.zshrc` の `mise activate` に任せる):

```sh
# Mise (non-interactive only; interactive shells use `mise activate` in .zshrc)
if [[ ! -o interactive ]] && [[ -z $MISE_GLOBAL_CONFIG_FILE ]]; then
  export MISE_GLOBAL_CONFIG_FILE="$HOME/dotfiles/.mise.toml"
  export PATH="$HOME/.local/share/mise/shims:$PATH"
fi
```

確認:

```bash
ssh jxck@jxck.io 'echo "PATH=$PATH"; echo "MISE_GLOBAL_CONFIG_FILE=$MISE_GLOBAL_CONFIG_FILE"; which git-lfs-transfer'
```

`git-lfs-transfer` が shim パスを返せば OK。

さらに protocol handshake が返ることまで確認する:

```bash
ssh jxck@jxck.io 'git-lfs-transfer /home/jxck/server/mozaic.fm.git upload </dev/null' 2>&1 | head
```

`version=1` を含む出力が返れば server 側 OK。

#### 4. bare repo を作る

```bash
git init --bare /home/jxck/server/mozaic.fm.git
git -C /home/jxck/server/mozaic.fm.git symbolic-ref HEAD refs/heads/main
```

LFS オブジェクト置き場の `lfs/objects/` は初回 push 時に自動で作られるので、事前作成はしない。bare 側 HEAD を `main` に向けておくことで、clone 直後から `main` ブランチにいる状態になる。

#### 5. 作業用 clone を作る

実体は `~/server/mozaic.fm` に置き、`~/develop/mozaic.fm` から symlink する (既存 `jxck.io` と同じ topology)。clone URL は最初から `ssh://` 形式を使う (LFS pure-SSH transfer のため)。

```bash
git clone ssh://jxck@jxck.io/home/jxck/server/mozaic.fm.git /home/jxck/server/mozaic.fm
ln -s /home/jxck/server/mozaic.fm /home/jxck/develop/mozaic.fm
cd /home/jxck/develop/mozaic.fm
git lfs install --local
git lfs track "files.mozaic.fm/**"
```

この時点で `.gitattributes` は `files.mozaic.fm/` 配下だけを LFS に送る設定になる。

#### 6. repo root のレイアウトを置く

```bash
mkdir files.mozaic.fm
```

`feed.mozaic.fm/` と `www.mozaic.fm/` は今は不要。Git は空ディレクトリを追跡しない。本格運用時に作ればよい。

#### 7. GitHub mirror remote を追加する

事前に GitHub 側に空 repo を作っておく (`gh repo create Jxck/mozaic.fm --private` 等)。

```bash
git remote add github git@github.com:Jxck/mozaic.fm.git
git config remote.github.lfsurl ssh://jxck@jxck.io/home/jxck/server/mozaic.fm.git
git config remote.github.lfspushurl ssh://jxck@jxck.io/home/jxck/server/mozaic.fm.git
```

`lfsurl` / `lfspushurl` も `ssh://` 形式で揃える。これらは `.git/config` にだけ置き、公開 repo に持ち出さない。

#### 8. 初回 commit を作る

```bash
git add .gitattributes
git commit -m "Initialize mozaic.fm repo with LFS layout"
git push -u origin main
git push -u github main
```

この段階ではまだ LFS オブジェクトが無くてもよい。まず repo の形だけを両 remote に載せる。

#### 9. `files.mozaic.fm/` 配下の mp3 1 本で `origin` の LFS を確認する

```bash
cp /home/jxck/server/files.mozaic.fm/mozaic-ep0.mp3 ./files.mozaic.fm/
git add files.mozaic.fm/mozaic-ep0.mp3
git commit -m "Add first MP3 via Git LFS"
git push origin main
```

確認点:

- `git lfs ls-files` に `files.mozaic.fm/mozaic-ep0.mp3` が `*` 付き (smudge 済み) で出る
- `/home/jxck/server/mozaic.fm.git/lfs/objects/<oid 先頭2桁>/<次の2桁>/<full oid>` に約 2 MB の実体ができる
- push 出力に `Uploading LFS objects: 100% (1/1)` が出る (legacy `git-lfs-authenticate` のエラーが出ていなければ pure-SSH transfer に乗っている)

#### 10. 同じ commit を GitHub mirror に送る

```bash
git push github main
```

期待する状態:

- GitHub には repo 全体の commit が載る (Git pack は数百 bytes 程度)
- LFS objects は `remote.github.lfsurl` 設定により VPS 側に upload 試行される (既存なので no-op)
- GitHub 側に mp3 実体は流れない
- GitHub Web UI で mp3 ファイルを開くと「Stored with Git LFS」と pointer (3 行) のみが表示される

#### 11. clone で復元確認する

`origin` 正本からの復元を確認する。

```bash
cd /tmp
git clone ssh://jxck@jxck.io/home/jxck/server/mozaic.fm.git verify
cd verify
git lfs pull
file files.mozaic.fm/mozaic-ep0.mp3
ls -lh files.mozaic.fm/mozaic-ep0.mp3
rm -rf /tmp/verify
```

`file` が音声ファイルとして認識し、サイズが元 mp3 と一致すれば成功。

### 注意点

- `github` mirror を clone しただけでは、他者は `files.mozaic.fm/` の mp3 を自動取得できない
- mp3 を取りたい自分の別マシン (Mac 等) からの clone では、(a) `git-lfs` と `git-lfs-transfer` を同様に揃え、(b) `origin` を VPS の `ssh://` URL に向ける必要がある
- これは「repo 全体の Git mirror は GitHub、mp3 実体は VPS only」という要件に沿った意図的な制約として扱う
- `git-lfs-transfer` は brew 管理外 (mise + go install) のため、build 失敗・breaking change には自前で追従が必要

### 完了条件

- `~/server/mozaic.fm.git` が bare repo として存在する
- `~/server/mozaic.fm` が repo root の実体として存在し、その下に `files.mozaic.fm/` を持つ
- `~/develop/mozaic.fm` が `~/server/mozaic.fm` への symlink として存在する
- non-interactive SSH から `git-lfs-transfer` が解決でき、`version=1` の handshake が返る
- `files.mozaic.fm/` 配下の mp3 1 本を `origin` に push すると LFS 実体が VPS 側に保存される (pure-SSH transfer 経由)
- 同じ commit を `github` に push でき、GitHub には pointer のみが載る
- `origin` から clone + `git lfs pull` で `files.mozaic.fm/` 配下の mp3 を復元できる
