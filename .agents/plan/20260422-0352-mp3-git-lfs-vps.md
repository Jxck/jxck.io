# Fixed Plan: ~/server/mozaic.fm に mp3 を LFS で置ける repo を作る

## tldr

- 正本は `~/server/mozaic.fm.git` (bare) と `~/develop/mozaic.fm` (clone) の 2 つで、`origin` は VPS 上の bare repo に固定する
- `~/develop/mozaic.fm` は repo 全体の作業木で、`feed.mozaic.fm` / `files.mozaic.fm` / `www.mozaic.fm` を同居させる
- Git LFS の対象は repo 全体ではなく `files.mozaic.fm/` 配下に限定し、実体は `~/server/mozaic.fm.git/lfs/objects/` に置く
- `github.com/Jxck/mozaic.fm` は repo 全体の Git mirror とし、GitHub の LFS は使わず local config で LFS だけ VPS に向ける
- 今回は repo 初期化、`files.mozaic.fm/` 配下の mp3 1 本で `origin` 疎通確認、`github` mirror への push 方針確立までを範囲とする

## Plan by Codex

### ゴール

以下の 3 点を同時に満たす構成を作る。

- `~/server/mozaic.fm.git`: bare repo。Git の正本 remote 兼 LFS オブジェクト保管先
- `~/develop/mozaic.fm`: 日常作業用 clone。repo root に `feed.mozaic.fm` / `files.mozaic.fm` / `www.mozaic.fm` を持つ
- `github.com/Jxck/mozaic.fm`: 公開用 mirror。repo 全体の commit は置くが、mp3 実体は持たせない

今回の範囲は「repo の器を作ること」と「`files.mozaic.fm/` 配下の mp3 1 本で transport を検証すること」までに留める。

### 範囲外

- `files.mozaic.fm/` への全 mp3 一括取り込み
- `feed.mozaic.fm/` / `www.mozaic.fm/` の中身の移植
- `jxck.io` 側 `mozaic.fm/` 実装の削除
- h2o 設定変更
- バックアップ運用の実装

### 設計方針

#### repo と remote の役割

- `~/develop/mozaic.fm` を repo root とする
- `origin = jxck@jxck.io:/home/jxck/server/mozaic.fm.git`
  - Git の push/pull の正本
  - LFS upload/download の正本
- `github = git@github.com:Jxck/mozaic.fm.git`
  - repo 全体の commit を載せる mirror
  - GitHub 側の LFS endpoint は使わない

#### LFS の source of truth

- LFS 対象は `files.mozaic.fm/` 配下だけに限定する
- `.gitattributes` もその前提で管理する
- `*.mp3` のような repo 全体指定にはしない

#### LFS endpoint の持ち方

- `.lfsconfig` は作らない
- `origin` は SSH remote から導出される pure-SSH transfer をそのまま使う
- `github` は `.git/config` の `remote.github.lfsurl` / `remote.github.lfspushurl` を VPS に向ける
  - これで `git push github main` 時も LFS 実体は VPS に送られる
  - 公開 repo 側には VPS の LFS endpoint を commit しない

#### server 側の前提

- bare repo 単体でよい
- ただし non-interactive SSH から `git-lfs` が見えることは必須
- 最初の LFS push 前に `which git-lfs` を確認する

### 想定レイアウト

```text
/home/jxck/server/mozaic.fm.git/
├── objects/
├── refs/
└── lfs/objects/

/home/jxck/develop/mozaic.fm/
├── .git/
├── .gitattributes
├── feed.mozaic.fm/
├── files.mozaic.fm/
├── www.mozaic.fm/
└── ...
```

### 手順

#### 1. `git-lfs` を入れる

```bash
brew install git-lfs
git lfs version
```

必要条件は「SSH transfer を使える `git-lfs` 3.x 系が入ること」であり、特定 patch version は前提にしない。

#### 2. bare repo を作る

```bash
git init --bare /home/jxck/server/mozaic.fm.git
```

LFS オブジェクト置き場の `lfs/objects/` は初回 push 時に自動で作られるので、事前作成はしない。

#### 3. 作業用 clone を作る

transport を `origin` と揃えるため、同一ホスト上でも SSH URL で clone する。

```bash
git clone jxck@jxck.io:/home/jxck/server/mozaic.fm.git /home/jxck/develop/mozaic.fm
cd /home/jxck/develop/mozaic.fm
git switch -c main
git lfs install --local
git lfs track "files.mozaic.fm/**"
```

この時点で `.gitattributes` は `files.mozaic.fm/` 配下だけを LFS に送る設定になる。

#### 4. repo root のレイアウトを置く

repo root には少なくとも以下を置く前提で進める。

- `feed.mozaic.fm/`
- `files.mozaic.fm/`
- `www.mozaic.fm/`

この phase では `files.mozaic.fm/` に mp3 1 本があれば十分で、他ディレクトリの実データ移植は範囲外とする。

#### 5. GitHub mirror remote を追加する

```bash
git remote add github git@github.com:Jxck/mozaic.fm.git
git config remote.github.lfsurl ssh://jxck@jxck.io/home/jxck/server/mozaic.fm.git
git config remote.github.lfspushurl ssh://jxck@jxck.io/home/jxck/server/mozaic.fm.git
```

この設定は `.git/config` にだけ置く。公開 repo に持ち出さない。

#### 6. non-interactive SSH から `git-lfs` が見えることを確認する

```bash
ssh jxck@jxck.io 'which git-lfs'
```

ここで解決できない場合は、そのまま進めない。Linuxbrew の PATH が login shell にしか入っていない可能性があるので、non-interactive SSH でも `git-lfs` が見える置き方に修正してから再開する。

#### 7. 初回 commit を作る

```bash
git add .gitattributes
git commit -m "Initialize mozaic.fm repo with LFS layout"
git push -u origin main
git push -u github main
```

この段階ではまだ LFS オブジェクトが無くてもよい。まず repo の形だけを両 remote に載せる。

#### 8. `files.mozaic.fm/` 配下の mp3 1 本で `origin` の LFS を確認する

```bash
cp /home/jxck/server/files.mozaic.fm/mozaic-ep0.mp3 ./files.mozaic.fm/
git add files.mozaic.fm/mozaic-ep0.mp3
git commit -m "Add first MP3 via Git LFS"
git push origin main
```

確認点:

- `git lfs ls-files` に `files.mozaic.fm/mozaic-ep0.mp3` が出る
- `/home/jxck/server/mozaic.fm.git/lfs/objects/` に実体ができる

#### 9. 同じ commit を GitHub mirror に送る

```bash
git push github main
```

期待する状態:

- GitHub には repo 全体の commit が載る
- `files.mozaic.fm/` 配下の LFS 実体の upload 先は引き続き VPS
- GitHub 側に mp3 実体を持たせない

#### 10. clone で復元確認する

`origin` 正本からの復元だけを確認する。

```bash
cd /tmp
git clone jxck@jxck.io:/home/jxck/server/mozaic.fm.git verify
cd verify
git lfs pull
file files.mozaic.fm/mozaic-ep0.mp3
```

`file` が音声ファイルとして認識すれば成功。

### 注意点

- `github` mirror を clone しただけでは、他者は `files.mozaic.fm/` の mp3 を自動取得できない
- mp3 を取りたい自分の clone では、同じ `remote.<name>.lfsurl` 設定を local に入れる必要がある
- これは「repo 全体の Git mirror は GitHub、mp3 実体は VPS only」という要件に沿った意図的な制約として扱う

### 完了条件

- `~/server/mozaic.fm.git` が bare repo として存在する
- `~/develop/mozaic.fm` が repo root として存在し、その下に `feed.mozaic.fm/` / `files.mozaic.fm/` / `www.mozaic.fm/` を持つ
- `files.mozaic.fm/` 配下の mp3 1 本を `origin` に push すると LFS 実体が VPS 側に保存される
- 同じ commit を `github` に push できる
- `origin` から clone + `git lfs pull` で `files.mozaic.fm/` 配下の mp3 を復元できる
