# Secret Key の運用とアカウントリカバリ

## 2FA の設定ルール

ビジネスアカウントにとって漏洩対策をすべきなのは、記憶に頼れない Secret Key の方だ。

個人のアカウントでは、これは Emergency Kit として発行され、そこに Master Password を記入して保管する方法が推奨されている。しかし、これは組織アカウントでは異なる。

Secret Key は、新しいデバイスでログインする際に必要で、それは既にログイン済みのデバイスで表示して入力すれば良い。つまり Secret Key を保管する必要があるのは、ログイン済みのデバイスが全てない状態、つまりリカバリ時なのだ。しかし、組織のアカウントにおいては、基本的に管理者がリカバリをして、メンバーのアカウントを復旧できるため、Secret Key の管理をメンバーに任せる必要がない。

したがって、メンバーは Secret Key を保管せず、リカバリが必要なときだけ管理者に依頼するという運用を取るのが、最も安全性が高いと言えるだろう。したがって、Emergency Kit のように雑に管理されるとリスクになるものは、最初から発行しないように設定することができる。

また、Secret Key だけで不安な場合、2FA を設定することが可能だ。個人の場合は、2FA も 1Password に寄せる運用になるため、1Password の 2FA を設定するには別途 Authenticator が必要になり、その 2FA はどう管理するかという堂々巡りな問題に発展する。

しかし、組織の場合は、情シス相当がしてする Authenticator が配布デバイスにインストールされ、アカウントも配られていることが多いだろう。仮に 2FA をロストしても、管理者のリカバリで復元することが可能なため、メンバーが 2FA できる環境が整っているなら、有効にすることで、より堅牢にメンバーのアカウントを保護することができる。
